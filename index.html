Module 1

Blockchain basics

A blockchain is a shared record book that no single person controls. Before diving into the cryptography, let’s understand the problem it solves.

The problem: trusting a record keeper

Banks, land registries and payment networks all work the same way: one trusted party keeps the official record. That works — until the record keeper makes a mistake, gets hacked, censors you, or simply isn’t trusted by everyone involved.

A blockchain flips the model: everyone keeps a full copy of the ledger, new entries are only added when the network agrees on them, and old entries are mathematically protected from editing.

Four properties to remember

Distributed
Thousands of computers (nodes) each hold the complete ledger.
Append-only
You can add new records, but never edit or delete old ones.
Tamper-evident
If anyone alters history, every other copy immediately exposes the change.
Consensus-driven
Rules — not a boss — decide which new entries are accepted.

Try it: the network rejects a cheat

Three nodes hold identical copies of a tiny ledger. Tamper with one and see what the majority does.

  • Alice → Bob: 5

  • Carol → Dan: 1

Node B can scribble whatever it wants in its own copy — but A and C still agree on the truth, so the network ignores B. To rewrite history you’d have to out-vote the entire honest network.

Where does cryptocurrency fit in?

A cryptocurrency is simply what the ledger tracks: entries like “Alice pays Bob 5 coins”. Because the ledger can’t be forged or rewritten, those entries behave like money — scarce, transferable, and not dependent on any bank. Bitcoin (2009) was the first system to make this work; thousands of others followed.

Module 2

Hashing — the fingerprint machine

Every security promise a blockchain makes traces back to one tool: the cryptographic hash function. Master this and the rest of the course clicks into place.

What a hash function does

A hash function takes any input — a letter, a novel, a 4 GB movie — and produces a fixed-size fingerprint. Bitcoin and this course use SHA-256, which always outputs 256 bits (written as 64 hex characters).

  • Deterministic — same input, same hash. Always.
  • One-way — there is no way to compute the input from the hash. Guessing is the only option.
  • Avalanche effect — change one character and the entire hash changes beyond recognition.
  • Collision-resistant — finding two inputs with the same hash is astronomically hard.

A one-character tweak flips roughly half of all 256 bits. That’s why a hash works as a tamper seal: even the tiniest edit is glaringly obvious.

Why blockchains care

Blockchains use hashes as tamper-evident seals. Hash a block’s contents and you get its fingerprint; change anything inside the block and the fingerprint no longer matches. In the next module you’ll see how chaining these fingerprints together protects an entire history.

Module 3

Blocks & the chain

A blockchain is literally a chain of blocks — and the links are hashes. Time to break one on purpose.

Anatomy of a block

Data
The payload — typically a batch of transactions.
Previous hash
The fingerprint of the block before this one. This is the chain link.
Nonce
A number miners adjust until the block’s hash meets the difficulty rule (more in Module 4).
Hash
The fingerprint of everything above. Here, a block is valid when its hash starts with 00.

Because each block embeds the previous block’s hash, the blocks form a chain: changing any old block changes its hash, which breaks the next block’s “previous hash”, which breaks the one after that — a domino effect all the way to the tip.

Try it: break the chain, then repair it

  1. Edit the data in Block #1 (change a name or an amount).
  2. Watch it and every later block turn invalid — the dominoes fall.
  3. Press Mine on each broken block, left to right, to repair the chain.

On a real network you’d have to redo this repair work faster than thousands of honest miners extending the chain — and that’s exactly why old blocks are effectively permanent.

Module 4

Mining & proof of work

Mining sounds mysterious, but you just did it in Module 3. It’s a guessing game — played at planetary scale.

The lottery puzzle

To add a block, a miner must find a nonce that makes the block’s hash start with a required number of zeros. Hashes are unpredictable, so there’s no shortcut: miners simply try nonce 0, 1, 2, 3… billions of times per second. Finding a winning nonce is hard; checking it takes anyone a single hash. That asymmetry is the magic.

  • Difficulty — each extra required zero multiplies the expected work by 16 (in hex). Networks auto-tune difficulty to keep block times steady.
  • Reward — the winning miner earns new coins plus fees. This is how new cryptocurrency enters circulation.
  • Security — rewriting history means redoing all that work and outpacing every honest miner. The energy cost is the lock on the vault.

Module 5

Keys, wallets & signatures

If hashes protect the history, digital signatures protect the ownership. Your “account” on a blockchain is nothing more than a key pair.

The key pair

  • Private key — a giant random number only you know. It is your identity. Whoever holds it controls the funds — there is no “forgot password” link.
  • Public key — derived from the private key and safe to share. Your address is (roughly) a hash of it.
  • Signature — a proof, computed with your private key, that you approved a specific message. Anyone can verify it with your public key; nobody can forge it without your private key.

A wallet doesn’t hold coins — coins only exist as ledger entries. A wallet holds your keys and signs transactions on your behalf.

Custody in one sentence

“Not your keys, not your coins” — if an exchange holds your private keys, you hold an IOU, not cryptocurrency. Hardware wallets and seed phrases exist purely to keep private keys safe and recoverable.

Module 6

Transactions & consensus

You now know every ingredient. Let’s follow a transaction end to end, then look at the two big ways networks agree on the next block.

Life of a transaction

  1. Create — your wallet drafts “Alice pays Bob 5”, referencing coins you own.
  2. Sign — your private key signs it (Module 5).
  3. Broadcast — it spreads node-to-node into the mempool, the waiting room of unconfirmed transactions.
  4. Confirm — a miner/validator packs it into a block (Modules 3–4). Each later block buries it deeper; ~6 confirmations is considered settled on Bitcoin.

Consensus rules also solve the double-spend problem: if you broadcast two transactions spending the same coin, the network will only ever confirm one — the other is rejected as invalid.

Beyond currency

Once a network can agree on arbitrary data, it can agree on code. Smart contracts are programs stored on-chain that execute exactly as written — powering tokens, decentralized exchanges, lending markets (DeFi) and NFTs. Different goals, same foundations you just learned.

Reference

Glossary

Quick definitions for the terms used across the course. One opens at a time.

Blockchain

A shared, append-only ledger replicated across many computers, where blocks of records are linked by hashes so history can’t be quietly rewritten.

Block

A batch of transactions plus metadata: the previous block’s hash, a nonce, and its own hash. The first block of a chain is the genesis block.

Hash

A fixed-size fingerprint of any data, produced by a one-way function such as SHA-256. Any change to the input produces a completely different hash.

Node

A computer that stores the ledger, validates new blocks and relays transactions. Anyone can run one.

Nonce

“Number used once” — the value miners vary to search for a hash that satisfies the difficulty target.

Mining

Racing to find a valid nonce for the next block. The winner earns newly issued coins plus transaction fees.

Proof of Work (PoW)

Consensus by computation: blocks are expensive to produce but cheap to verify, so rewriting history costs more than honest participation.

Proof of Stake (PoS)

Consensus by collateral: validators lock up coins for the right to propose blocks, and lose them (“slashing”) if they cheat.

Private key

A secret number that controls an address’s funds. Whoever knows it owns the coins; lose it and the coins are gone forever.

Public key / address

The shareable half of a key pair. An address — what you give people to get paid — is essentially a hash of the public key.

Digital signature

Proof, verifiable by anyone, that the holder of a private key approved one exact message. The basis of transaction authorization.

Wallet

Software or hardware that stores keys and signs transactions. It holds no coins — coins exist only as ledger entries.

Mempool

Each node’s waiting room of valid but not-yet-confirmed transactions, from which miners/validators assemble blocks.

Confirmation

Each block added after the one containing your transaction. More confirmations = deeper burial = harder to reverse.

Double-spend

Trying to spend the same coin twice. Consensus guarantees at most one of the conflicting transactions is ever confirmed.

51% attack

Controlling a majority of mining power (PoW) or stake (PoS), enough to censor or reorder recent transactions. Prohibitively expensive on large networks.

Smart contract

A program stored on-chain that runs exactly as written when triggered by a transaction — the engine behind tokens, DeFi and NFTs.

Gas / fees

The price paid to have the network process your transaction or run your contract; it compensates miners/validators and prevents spam.

Token vs. coin

A coin is a blockchain’s native currency (BTC, ETH). A token is an asset created by a smart contract on top of an existing chain.

DeFi & NFT

Decentralized finance: lending, trading and earning interest via smart contracts. NFTs: unique on-chain tokens representing ownership of a specific item.
